### 1. Vision Parser — detected vs drawn

**Hand-drawn run** (`dashboard_handdrawn/run.txt`):

- **screen_title:** Admin Dashboard
- **components (9):** `heading:Header`, `nav:Sidebar`, `text:Side bar`, `card:stat`, `text:stat`, `card:stat`, `text:stat`, `container:Image`, `text:Image`
- **notes:** Correctly identifies sidebar, header, two stat cards, and image area; notes handwriting is hard to read precisely.

**Synthetic baseline** (`dashboard/run.txt`):

- **screen_title:** Dashboard
- **components (11):** `heading:Dashboard`, `nav:Sidebar Navigation`, `text:Home`, `text:Settings`, `text:Reports`, `card:Revenue`, `text:Revenue`, `text:$12,400`, `card:Users`, `text:Users`, `text:1,284`

| Category | Assessment |
|----------|------------|
| **Layout regions (correct)** | Header, sidebar, two stat cards, and image container all detected on the hand-drawn photo. Structure match is strong. |
| **Handwriting OCR (correct)** | "Side bar", "stat" (×2), "Image" read correctly. "He ader" interpreted as `heading:Header` (reasonable normalization). |
| **Handwriting OCR (miss)** | No explicit capture of the split "He ader" spelling; title upgraded to "Admin Dashboard" (description influence). |
| **Hallucinated content** | None at vision stage. Vision stuck to what is on the page. |
| **Downstream fill-in** | Architect/Code Forge added nav labels (Dashboard, Users, Settings) and metric values ($12,345, 1,234) **not present on the photo** — driven by description string + generic dashboard patterns, not vision errors. |

**Verdict:** Vision accuracy does **not** drop meaningfully vs synthetic for layout. The hand-drawn sketch has less literal text to read (generic "stat" labels vs specific dollar amounts), so downstream agents invent content. That is a content mismatch, not an OCR failure.
